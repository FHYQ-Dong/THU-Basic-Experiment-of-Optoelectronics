# Alice 主机端

QKD-BB84 实验中 Alice（发送方）的主机端代码与数据。
负责生成随机偏振序列、通过串口送给 FPGA 驱动激光器，
同时把真值序列存盘，供 Bob 端解码后做基比对。

---

## 1. 文件清单

| 文件 / 目录             | 作用                                                                                    |
| ----------------------- | --------------------------------------------------------------------------------------- |
| `random-gen.py`         | 生成 `0~3` 的随机字节流，通过串口连续发给 FPGA，最后发 `b'q'` 停机；同时把真值序列存为 txt |
| `data/`                 | 实验过程中产生的数据（TCSPC 输出、随机序列真值）                                          |
| `data/random_data_*.txt`| Alice 发送过的随机序列真值，一行一个数字 (0/1/2/3)。Bob 端 `compare_channel_with_txt.py` 读取它做基比对 |
| `data/TCSPC_*.txt`      | FT1080 单光子计数器在该次实验中导出的原始事件流（同步信号 + 单光子事件）                  |

---

## 2. 协议约定

字节的低 2 位编码偏振信息（与 FPGA `laser_ctrl.v` 的独热映射、
报告表格中的「发送态编号」全部保持一致）：

| 字节值 | 激活激光器 | 偏振态        | 基（0=直角 / 1=对角）|
| ------ | ---------- | ------------- | -------------------- |
| `0x00` | `laser[0]` | H（0°）       | 0                    |
| `0x01` | `laser[1]` | V（90°）      | 0                    |
| `0x02` | `laser[2]` | +45°          | 1                    |
| `0x03` | `laser[3]` | -45°          | 1                    |
| `0x71` | —          | —             | 停止信号（ASCII `'q'`）|

> Bob 端探测器的物理通道编号与 Alice 端的发射通道不一致（光路对称性 +
> 接线决定），在 `Bob/extract_unique_events_to_csv.py` 和
> `extract_effective_events_from_t2_csv.py` 中通过 `CHANNEL_MAP = {0:0, 1:3, 2:2, 3:1}`
> 把 Bob 端原始通道映射回 Alice 的偏振索引，便于后续直接比对。

## 3. 运行环境

| 工具                | 版本/型号                |
| ------------------- | ------------------------ |
| Python              | 3.12（项目根 `.python-version`）|
| `pyserial`          | 见根目录 `pyproject.toml`  |
| 串口                | `COM8`（脚本里硬编码，需要按实际板子修改）|
| 波特率              | 250 000 bps（与 FPGA `top.v` 默认值一致）|
| Windows-only        | 脚本结束时调用 `winsound.Beep` 提示，Linux/macOS 上需要注释掉 |

## 4. 典型流程

1. 用 Vivado / 开源工具链把 `FPGA-control/` 烧到 FPGA，FPGA 处于等待 UART 输入的状态；
2. 修改 `random-gen.py` 中的 `run_time`（秒）和文件名后缀，
   `total_numbers = 250000 / 10 * run_time` 即「波特率 / 单字节帧长」× 时长，
   等于 FPGA 同步频率（50 kHz）× 时长，与同步周期一一对应；
3. 在 Python 环境中执行：

   ```bash
   cd QKD-BB84
   python Alice/random-gen.py
   ```

4. 脚本会：
   - 一次性生成 `total_numbers` 个 `[0, 3]` 范围的随机整数；
   - 转成字节流通过串口连续发出；
   - 紧跟着发一字节 `b'q'`，让 FPGA 进入 `S_STOPPED`；
   - 把整段随机序列写入 `random_data_<duration>s_<baud>.txt`（一行一个数字）；
   - 蜂鸣三声提示结束。

5. 与此同时，Bob 端 TCSPC 软件应已在记录同步信号 + 4 路单光子事件，
   实验结束后导出原始 txt 文件，进入 `Bob/` 的解码流程。

---

## 5. 注意事项

- **不要把 `random_data_*.txt` 与 FPGA 输出混在一起**——前者是 Alice
  端生成的真值序列，后者是 Bob 端探测到的事件流。基比对脚本会依赖这两者
  的「行序号 vs. 事件序号」严格对齐，所以两次实验之间最好用时长 + 日期
  作为文件名后缀加以区分。
- 末尾几行注释（`# 0 -> CH1 -> 0` 等）记录的是某次校准时各通道的实测光
  功率（μW）以及通道映射表，调试用，不影响功能。
- 脚本一次性把所有字节拼好再发，250 kbps 下 1800 秒的实验会发 45 MB，
  `ser.write` 会阻塞直到 OS 缓冲区写完，这就是 `elapsed_time` 的来源；
  并不代表实际光子发射节奏（实际节奏由 FPGA 的同步信号决定）。
